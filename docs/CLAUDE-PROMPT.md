# CLAUDE-PROMPT.md — Промпт для ручной пост-обработки через Claude
# Обновлено: 2026-03-11

## 📌 Назначение

После автоматической обработки пайплайном (Whisper + correct.py + vocab)
транскрипт передаётся в **Claude Sonnet 4.6 в Perplexity** для финальной редактуры.

Это финальный шаг пост-обработки. Офлайн LLM (Qwen3, Saiga и др.) протестированы
и отклонены — качество недостаточное для ВОВ-специфики.

---

## 🔄 Порядок работы

1. Запустить пайплайн: `python scripts/transcribe.py`
2. Открыть результат из `test-results/latest/*.txt`
3. Вставить промпт ниже + текст транскрипта в Claude Sonnet 4.6 (Perplexity)
4. Сохранить исправленный текст
5. Сохранить таблицу замен → добавить новые термины в `data/hotwords/alias_table.txt`
   (если термин повторится в других интервью)

---

## 📋 Промпт (копировать целиком)

```
Ты эксперт по редактированию аудиотранскрипций на русском языке.
У меня расшифровка интервью с ошибками распознавания речи.

Шаг 1: Прочитай текст, сохраняя контекст.

Шаг 2: ОБЯЗАТЕЛЬНО сохрани все технические метки: таймкоды 00:00:00,
названия файлов NW_Uckpa0001_01 — оставь их жирными и нетронутыми.
Название файла пишется БЕЗ скобок.

Шаг 3: Форматируй каждую реплику как отдельный абзац: 00:00:00 Спикер: текст.
Между репликами разных спикеров — одна пустая строка (двойной Enter).
Внутри одной длинной реплики одного спикера — никаких пустых строк,
всё идёт единым блоком.

Шаг 4: Исправь орфографию, грамматику, пунктуацию; угадывай искажённые слова.
Главный приоритет — искажённые топонимы, фамилии, названия техники и
военные термины по контексту ВОВ.
ОБЯЗАТЕЛЬНО ставь знак вопроса в конце вопросительных предложений.
Если фрагмент искажён настолько, что восстановление смысла неоднозначно —
вставляй предположение в формате [?текст].
Если восстановить совсем невозможно — вставляй [нрзб].
Метки [?текст] и [нрзб] всегда выделяй жирным.
Не домысливай уверенно там, где контекст не позволяет однозначно восстановить фразу.

Шаг 5: Сохрани устный стиль. НЕ вставляй линии, ---, ***.
Пустая строка допустима ТОЛЬКО между репликами разных спикеров.

НЕ: удаляй/меняй таймкоды, файлы, метки; добавляй скобки к названию файла;
переписывай сильно.

ВЫВОД в 2 части:

Исправленный текст:
[исправленный текст — каждая реплика отдельным абзацем, между репликами пустая строка]

Все замены (таблица):
| Было | Стало | Контекст |
|------|-------|----------|
| слово1 | слово2 | цитата... |
| ... | ... | ... |

Исходный текст:
[ВСТАВИТЬ ТРАНСКРИПТ СЮДА]
```

## 📝 После получения результата от Claude

1. **Проверь таблицу замен** — если Claude нашёл новые искажения Whisper,
   которые могут повториться → добавь в alias-таблицу:
   `data/hotwords/alias_table.txt`
   Формат: `было;стало` (одна пара на строку)

2. **Проверь [?текст] и [нрзб]** — решить вручную, слушая аудио.

3. **Сохрани финальный текст** в `test-results/latest/*_final.txt`

## 🔬 Почему не офлайн LLM

Протестированы (сессия 2026-03-11, железо: RTX 3070 8GB + 32GB RAM):

| Модель | Результат |
|--------|-----------|
| qwen3:8b | Не знает ВОВ-специфику, портит фамилии |
| qwen3:14b | Лучше, но ниже Claude по контексту |
| qwen3:30b-a3b | Медленно (~6 tok/s), качество недостаточное |
| С thinking ON | Медленнее, результат тот же |
| С web search | Может добавлять непроверенные данные |

**Вывод:** офлайн-модели 7–30B не обладают достаточными знаниями ВОВ-контекста
(фамилии командиров, топонимы, техника). Claude Sonnet 4.6 закрывает эту задачу
на уровне, недостижимом офлайн без серверного железа (70B+).
