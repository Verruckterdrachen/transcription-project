### BAG_D_v3 — таймкод внутри предложения после auto_merge
- **Статус:** ✅ FIXED v17.18
- **Файл:** NW_Uckpa0001_01 → Исаев (27.05).txt
- **Таймкод:** 00:23:37
- **Симптом:** «...броня до 100 мм, лоб, который был практически  00:23:37  непоражаем...» — ts внутри предложения
- **Root cause:** `merge_replicas` остановился на паузе 2.6s (порог 2.0s) → блок `00:23:07–00:23:51` (44s) получил FALLBACK inject на word#46/71. После этого `auto_merge` склеил блоки, но повторный `insert_intermediate_timestamps` сделал SKIP (хвост 12s ≤ 45s) — не очистив старые ts перед повторным проходом
- **Этап пайплайна:** insert_intermediate_timestamps → auto_merge → insert_intermediate_timestamps (повторный)
- **Фикс:** перед повторным вызовом `insert_intermediate_timestamps` в этапе 9 — очищаем все ts из текстов сегментов через `re.sub`
- **Файл фикса:** `scripts/transcribe.py`
- **Исправлено в версии:** v17.18
- **Как найти быстрее:** `grep "AFTER AUTO-MERGE\|── Сегмент" debug.log` — если после AUTO-MERGE нет строк `📌 inject` для длинных блоков → повторный inject был пропущен через SKIP
